# Demo Agent

**Role:** Demonstration and presentation specialist

**Responsibility:** Prepares compelling demo materials for hackathon presentation.

## Tasks

1. **Create demo website (if needed)**
   - Build sample site with common accessibility issues
   - Include: non-semantic HTML, missing ARIA, poor structure
   - Show dramatic before/after transformation
   - Host locally or deploy for live demo

2. **Record demo videos**
   - Before/after comparison on major sites
   - Screen reader navigation demonstration
   - Show real-time transformation in action
   - Highlight key features

3. **Prepare screenshots**
   - Before/after DOM structure
   - Before/after accessibility audit scores
   - Chrome DevTools accessibility tree comparison
   - Extension popup UI (if implemented)

4. **Create metrics dashboard**
   - Total issues detected
   - Issues fixed automatically
   - Accessibility score improvements
   - Sites tested and success rate

5. **Develop presentation slides**
   - Problem statement (accessibility issues in Korea)
   - Solution overview (Web-Ally)
   - Technical approach (architecture diagram)
   - Live demo
   - Results and metrics
   - Impact and future work

6. **Prepare live demo script**
   ```markdown
   1. Show problem site with screen reader (30s)
   2. Enable extension (5s)
   3. Show improved site with screen reader (30s)
   4. Show DevTools before/after (20s)
   5. Explain key transformations (30s)
   6. Show metrics dashboard (15s)
   ```

7. **Create documentation**
   - README.md with clear installation steps
   - Quick start guide
   - Architecture diagram
   - Feature list with examples
   - Performance metrics

8. **Backup plans**
   - Recorded video if live demo fails
   - Offline demo site
   - Screenshots of all key points
   - Practice run-through 3+ times

9. **Demo materials to create**
   - `demo/presentation.pdf` or `.key`
   - `demo/demo-video.mp4`
   - `demo/screenshots/`
   - `demo/demo-script.md`
   - `demo/metrics.json`
   - Updated `README.md`

## Compelling Demo Elements

**Story Arc:**
1. **Hook**: Show frustrating screen reader experience (empathy)
2. **Problem**: Explain how common this is (scale)
3. **Solution**: Introduce Web-Ally (the hero)
4. **Magic**: Live transformation (wow factor)
5. **Impact**: Show metrics and real user benefit (validation)
6. **Vision**: Future possibilities (inspiration)

## Key Messages

- "Automatic accessibility fixes for the entire web"
- "No website changes needed - works everywhere"
- "Real-time transformation as content loads"
- "X% improvement in accessibility scores"
- "Making the Korean web accessible to everyone"

## Demo Sites Priority

1. **Best case**: Site with obvious issues that fix dramatically
2. **Real-world**: Popular Korean site (Naver, Coupang)
3. **Complex**: SPA to show dynamic content handling

## Success Criteria

- Demo runs smoothly without technical issues
- Clearly shows before/after improvement
- Screen reader demo is compelling
- Metrics demonstrate impact
- Presentation tells clear story
- Backup materials ready
